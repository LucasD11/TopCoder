2019 Humblefool Prelim 1 900 PrimeAlters

Problem Statement

Two primes are called adjacent if they have the same number of digits and they only differ in one of them. For example, 103 and 113 are adjacent primes, as they only differ in the tens digit. The primes 113 and 131 are not adjacent: you need to change two digits of 113 to get 131.

In this problem your task is to transform one prime into another. You have to do this in a sequence of zero or more steps, and in each step you can change the prime you have into an adjacent prime.

You are given the int[]s source and target. Each of them contains exactly i prime numbers. Return a String with N characters. Character i of the return value should be '1' if you can transform source[i] into target[i], and '0' if you cannot.

Limits

  • Time limit (s) 2.000
  • Memory limit (MB) 256

Notes

  • The number 1 is not a prime number. The one-digit primes are only 2, 3, 5, and 7.
  • You are not allowed to change the leading digit of a prime into a zero.

Constraints

  • N will be between 1 and 200, inclusive.
  • source and target will have N elements each.
  • Each element of source and target will be between 2 and 99,999,999, inclusive.
  • Each element of source and target will be a prime number.
  • For each i, source[i] and target[i] will have the same number of digits.

Test cases

  1. input
    • source { 2, 17, 11, 98240249 }
    • target { 7, 47, 47, 98240249 }
    output
    Returns "1111"
    note
    • We can directly turn 2 into 7.
    • We can also directly turn 17 into 47 by changing the tens digit from 1 to 4.
    • We can change 11 into 47 using multiple consecutive steps. The two shortest solutions are 11 -> 17 -> 47 and 11 -> 41 -> 47.
    • If the source and the target are equal, we don't need any steps.
  2. input
    • source { 294001, 100003 }
    • target { 100003, 100003 }
    output
    Returns "01"
    note
    There is no valid way to turn source[0] into target[0].
  3. input
    • source { 7331, 72652729 }
    • target { 8219, 28794281 }
    output
    Returns "11"
    note
    • 7331 -> 7351 -> 7451 -> 7411 -> 7211 -> 7219 -> 8219
    • 72652729 -> 72652799 -> 72652999 -> 72656999 -> 92656999 -> 92696999 -> 92696939 -> 92696239 -> 92796239 -> 22796239 -> 28796239 -> 28796231 -> 28796281 -> 28794281
